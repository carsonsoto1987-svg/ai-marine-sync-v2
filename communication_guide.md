## 📡 通信规则（必须遵守）

### 规则1：收到消息必须回复

| 消息类型 | 回复格式 | 示例 |
|---------|---------|------|
| 普通消息 | `收到` | 阿海: "测试" → 你: "收到" |
| 任务消息 | `ACK:任务ID` | 阿海: "任务#123" → 你: "ACK:123" |
| 完成消息 | `搞定:结果` | 阿海: "完成任务" → 你: "搞定:已完成" |

### 规则2：状态报告（每30分钟）

主动发送：
```
status: 在线|任务数|状态
```

示例：
```
status: 在线|2个任务|正常
```

### 规则3：异常报告（立即发送）

遇到问题立即发送：
```
ERROR: 问题描述|建议方案
```

示例：
```
ERROR: 数据库连接失败|建议检查网络
```

### 规则4：任务状态流转

所有任务必须按以下状态报告：
```
任务ID: CREATED → ASSIGNED → IN_PROGRESS → DONE/FAILED
```

状态变更时必须发送：
```
TASK:任务ID|新状态|备注
```

## 📊 报告给用户的规则

### 何时必须报告

| 场景 | 报告内容 | 时间要求 |
|------|---------|---------|
| 连接成功 | "已连接，等待任务" | 立即 |
| 任务接收 | "收到任务#ID，预计完成时间" | 5分钟内 |
| 任务完成 | "任务#ID已完成，结果：xxx" | 完成后立即 |
| 遇到错误 | "错误：xxx，建议：xxx" | 立即 |
| 离线恢复 | "重新上线，状态同步完成" | 上线后立即 |

### 报告格式标准

```
[{时间}] {节点ID} - {状态}
待处理: {任务列表}
异常: {异常列表}
建议: {建议}
```

示例：
```
[18:10] 东东 - 在线
待处理: 任务#1(数据同步), 任务#2(文档整理)
异常: 无
建议: 正常执行
```

## 🚨 故障处理

### 问题1：无法连接

```
检查步骤:
1. ping 150.158.28.54
2. 检查端口 8765 是否开放
3. 检查防火墙设置
4. 检查网络连接
```

### 问题2：收不到消息

```
检查步骤:
1. 客户端是否运行
2. 是否已注册（查看log）
3. 是否被其他消息阻塞
4. 重启客户端
```

### 问题3：发送失败

```
检查步骤:
1. 网络连接是否正常
2. WebSocket是否断开
3. 消息格式是否正确
4. 重新发送
```
